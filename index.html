Creating an embedded widget

Using the modern JS toolchain

Presentation by Emil Stenqvist / @svammel

What's the deal?

Well, why roll your own when there are services like:

  1. Google Analytics
  2. Facebook buttons
  3. Get Satisfaction
  4. The Shootitlive player

These are all 3rd party widgets

Challenges

  1. Each embed needs its own settings
  2. Don't stop the DOM rendering - async loading
  3. Stay sane as a programmer - RequireJS modules
  4. Use your peers - 3rd party libs
Coding a widget is different form writing Javascript that will be the only scripts on a site.

Embed code exhibit

They come in all shapes and lengths.

The short one


<script src="//awesome-service.com/init.js"></script>
<script>
fireItUp({
  // Parameters
  project: 13, client: 'greenfield'
});
</script>
First thing people do, downsides:
  1. Stops parsing of whole site
  2. Little flexibility

The long one


<div id="your-unique-identifier"></div>
<script>
(function() {
  var s = document.createElement('script'); 
  s.src = '//latest-cdn.com/widget/master/main.js'; 
  s.async = true;
  if(!window.widget_options) {
      window.widget_options = [];
  }
  window.widget_options.push({
      special_setting: 123, project: 13, client: 'greenfield',
      element_id: 'your-unique-identifier', background: '#000' 
  });
  document.body.appendChild(s);
}());
</script>
